# RQTDW© 샘플 D·W 단계 확장 설명

## 📌 업데이트 요약

기존 샘플 1, 2에 **Discussion(D)과 Write(W) 단계**를 추가하여 **완전한 5단계 여정**을 체험할 수 있도록 확장했습니다.

---

## 📘 샘플 1: 창업 확신 고민

### 입력 질문
> "창업 아이디어가 확신이 안 서는데, 어떻게 해야 할까요?"

### 전체 여정

#### 🔶 Q (Question): 질문 재구성
**15창의질문 기법 적용** - 4가지 재구성된 질문

| 카테고리 | 질문 | 이유 |
|---------|------|------|
| Why 질문 | "왜 확신이 필요하다고 생각하나요?" | 확신의 필요성 자체를 질문하면 본질이 보입니다 |
| 가정 질문 | "확신이 있어야만 창업을 할 수 있다는 전제가 맞나요?" | 많은 성공한 창업가도 처음엔 확신이 없었습니다 |
| 관점 질문 | "고객 입장에서 보면, 당신의 확신이 중요한가요?" | 고객은 당신의 확신이 아니라 문제 해결을 원합니다 |
| 연결 질문 | "확신 대신 무엇이 있으면 시작할 수 있나요?" | 확신보다 중요한 것은 검증 가능한 작은 시작입니다 |

---

#### 🟢 T (Think): Head·Heart·Body 프레임

**사고 구조화**

```
Head (생각):
논리적으로는 '확신'이 아니라 '검증'이 필요하다는 걸 알게 됨.
MVP를 만들어 고객 반응을 보는 게 먼저.

Heart (감정):
불안하지만, 완벽하지 않아도 시작할 수 있다는 생각에 용기가 생김.
실패해도 배울 수 있다는 마음.

Body (행동):
이번 주 안에 고객 10명과 대화하기.
간단한 프로토타입 스케치 시작.
```

**카드 미리보기:**
- **Head:** '검증'이 필요, MVP로 테스트
- **Heart:** 불안하지만 용기 생김
- **Body:** 이번 주 고객 10명 대화

---

#### 🟣 D (Discussion): 질문 기반 토론

**3명의 참여자가 질문으로 대화**

| 참여자 | 질문 | 질문 타입 |
|-------|------|----------|
| **A** | "확신이 없다는 건, 혹시 아이디어 자체보다 '실패에 대한 두려움'이 더 큰 건 아닐까요?" | 전제 의심 |
| **B** | "만약 확신이 100%라면, 그게 오히려 위험 신호일 수도 있지 않을까요? 시장을 제대로 본 게 맞나요?" | 역발상 질문 |
| **C** | "지금 필요한 건 '확신을 얻는 것'일까요, 아니면 '확신 없이도 움직이는 법을 배우는 것'일까요?" | 재정의 질문 |

**핵심 인사이트:**
> "확신 없이도 움직이는 법을 배우는 게 목표 아닐까?"

---

#### ⚪ W (Write): 통찰 기록

**1. 핵심 통찰**
```
'확신'을 기다리는 것 자체가 함정이었습니다. 
진짜 필요한 건 작은 검증을 반복하며 배우는 과정입니다.
```

**2. 질문의 진화**

| 구분 | 질문 |
|-----|------|
| **처음 질문** | "창업 아이디어가 확신이 안 서는데, 어떻게 해야 할까요?" |
| **최종 질문** | "확신 대신 어떤 작은 실험으로 고객의 진짜 문제를 검증할 수 있을까?" |

**3. 실행 계획**
```
1) 이번 주 고객 10명 인터뷰
2) 가설 3가지 세우기
3) 가장 간단한 MVP 스케치
4) 2주 안에 첫 검증 완료
```

**4. 사고 레벨 리포트**

| 지표 | 점수 | 설명 |
|------|------|------|
| 질문 깊이 | 8.5/10 | 표면 → 본질 질문으로 발전 |
| 관점 다양성 | 7.5/10 | 자신/고객 관점 확보 |
| 구조화 능력 | 8.0/10 | Head·Heart·Body 활용 우수 |

## 📕 샘플 2: 팀 협업 문제

### 입력 상황
> "우리 팀은 회의는 많은데 결정은 안 되고, 각자 다른 방향으로 일하는 것 같아요. 소통은 많이 하는데 왜 협업이 안 되는 걸까요?"

### 전체 여정

#### 🔶 Q (Question): 질문 재구성

| 카테고리 | 질문 | 이유 |
|---------|------|------|
| 가정 질문 | "'협업'의 정의가 팀원마다 같다고 확신하나요?" | 같은 단어를 다르게 이해하면 협업은 불가능합니다 |
| 구분 질문 | "소통과 협업은 같은 것인가요?" | 말은 많지만 같은 목표를 향하지 않을 수 있습니다 |
| Why 질문 | "우리 팀의 공통된 Why는 무엇인가요?" | 목표가 정렬되지 않으면 방향이 제각각입니다 |
| 재정의 질문 | "이 문제를 '협업 부족'이 아니라 다르게 정의하면?" | 문제를 재정의하면 해결책이 달라집니다 |

---

#### 🟢 T (Think): 문제 재정의 맵

**3단계 문제 탐색**

```
표면 문제:
회의가 많고 소통이 많다

↓

근본 문제:
공통된 목표(Why)가 없어서 각자 다른 방향으로 일함

↓

재정의된 문제:
'협업 부족' 문제가 아니라 'Why가 정렬되지 않은' 문제
```

**카드 미리보기:**
- **표면:** 회의 많고 소통 많음
- **근본:** 공통 Why 없음
- **재정의:** Why 정렬 문제

---

#### 🟣 D (Discussion): 질문 기반 토론

| 참여자 | 질문 | 질문 타입 |
|-------|------|----------|
| **A** | "우리가 '회의'라고 부르는 게 진짜 의사결정 회의일까요, 아니면 그냥 정보 공유일까요?" | 구분 질문 |
| **B** | "만약 팀원 각자에게 '우리 팀의 최종 목표가 뭐예요?'라고 물으면 같은 답이 나올까요?" | 가정 질문 |
| **C** | "소통이 많다는 게 좋은 신호일까요, 나쁜 신호일까요? 혹시 목표가 불명확해서 계속 확인하는 건 아닐까요?" | 관점 전환 |

**핵심 인사이트:**
> "팀원마다 '최종 목표'를 물으면 같은 답 나올까?"

---

#### ⚪ W (Write): 통찰 기록

**1. 핵심 통찰**
```
문제는 '협업 부족'이 아니라 'Why의 부재'였습니다. 
소통은 많지만 방향이 없었던 겁니다.
```

**2. 질문의 진화**

| 구분 | 질문 |
|-----|------|
| **처음 질문** | "우리 팀은 회의는 많은데 결정은 안 되고, 각자 다른 방향으로 일하는 것 같아요." |
| **최종 질문** | "우리 팀이 함께 달성하려는 하나의 Why는 무엇이고, 그걸 어떻게 정렬할 수 있을까?" |

**3. 실행 계획**
```
1) 팀 전체 워크샵: 우리의 Why 정의하기
2) 의사결정 기준 3가지 합의
3) 회의 유형 구분: 정보공유 vs 의사결정
4) 주간 목표 정렬 체크인 도입
```

**4. 사고 레벨 리포트**

| 지표 | 점수 | 설명 |
|------|------|------|
| 질문 깊이 | 9.0/10 | 문제를 근본적으로 재정의함 |
| 관점 다양성 | 8.5/10 | 개인/팀/목표 관점 통합 |
| 구조화 능력 | 7.5/10 | 문제 재정의 맵 활용 |

## 🎨 샘플 카드 UI 개선

### Before (Q, T만 표시)
```
[Q] 재구성된 질문 예시
[T] 사고 프레임
```

### After (Q, T, D, W 모두 표시)
```
[Q] 재구성된 질문 예시
    • 4가지 질문 타입

[T] 사고 구조화
    • Head·Heart·Body / 문제 재정의 맵

[D] 질문 기반 토론
    • 핵심 토론 질문 1개

[W] 핵심 통찰
    • 최종 깨달음 요약
```

## 🎯 사용자 가치

### 1. 완전한 여정 미리보기
- 샘플 카드에서 R-Q-T-D-W 전체 과정을 한눈에 확인
- "이 시스템이 어떻게 작동하는가" 명확히 이해

### 2. 단계별 성과 예시
- **Q**: 어떻게 질문이 재구성되는가
- **T**: 어떻게 사고가 구조화되는가
- **D**: 어떻게 토론이 진행되는가
- **W**: 어떤 통찰과 성장이 생기는가

### 3. 실전 적용 가능성
- 실제 데이터 기반 샘플
- 구체적인 실행 계획 제시
- 사고 레벨 측정 예시
